# Task #19 Plan: Feature Flags Admin page

## Summary
Create a simple admin page at `/admin/feature-flags` for toggling feature flags on/off, following the PortfolioAreas.tsx pattern.

## Files to Create

### 1. `packages/frontend/src/pages/FeatureFlagsAdmin.tsx` (NEW)

**Structure** (following PortfolioAreas.tsx pattern):
- Header with title + subtitle (no create button — flags are seeded, not user-created)
- Table showing all flags with columns: Key, Description, Status (toggle), Updated
- Toggle switch for enabling/disabling each flag using `useUpdateFeatureFlag()`
- Uses `useFeatureFlags()` hook from Task #18

**No modals needed** — toggling is a simple inline switch, not a form operation.

## Files to Modify

### 2. `packages/frontend/src/router.tsx`
- Add lazy import for `FeatureFlagsAdmin`
- Add route at `admin/feature-flags` with `ErrorBoundary` + `LazyPage` wrapper

## Design
- ADMIN-only access is enforced by backend (`PUT` requires ADMIN role) and by nav visibility (Layout only shows link for ADMIN)
- Toggle uses `useUpdateFeatureFlag()` mutation with `{ enabled: !current }` — optimistic feel via React Query invalidation
- Loading state: show "Loading..." placeholder while flags load
- Empty state: unlikely but handled ("No feature flags configured")
